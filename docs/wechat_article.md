# Claw AI Lab：一句话启动一个 AI 研究团队

**From Prompt to Paper — 全自动多智能体科研平台**

---

> 想象一下：你输入一个研究想法，AI 自动帮你查文献、设计实验、写代码、跑实验、分析结果、写论文。这不是科幻，这是 **Claw AI Lab** 正在做的事。

---

## 一、它是什么？

**Claw AI Lab** 是一个面向科研场景的多智能体自动化平台。和传统的 AI 编程助手不同，它不只是帮你写代码——它模拟的是一个**完整的研究团队**：有人负责文献调研，有人负责实验设计，有人写代码跑实验，有人撰写论文，还有"审稿人"做质量把关。

**一个 Prompt，一个完整的研究闭环。**

---

## 二、26 步全自动流水线

Claw 的核心是一条贯穿「想法 → 论文」全生命周期的 **26 阶段流水线**，分为五大阶段：

### Phase 1 · 文献研究（S1-S7）

系统自动将你的研究课题分解为子问题，制定搜索策略，从 Semantic Scholar、arXiv 等学术数据库批量检索论文，经过相关性筛选后抽取结构化知识，最终生成文献综合分析。

### Phase 2 · 实验设计（S8-S10）

基于文献洞察自动生成可检验的研究假设，输出严谨的实验方案（`exp_plan.yaml`），包含实验条件、评估指标、计算预算等完整设计。同时搜索相关开源代码库作为参考。

### Phase 3 · 代码与执行（S11-S15）— **Claw-Code Harness 的核心**

这是 Claw 最具技术含量的部分：

- **S11 代码生成**：LLM 在沙箱中通过 **Turn Loop**（工具调用循环）迭代编写实验代码。它能读取你的本地数据集、模型检查点和代码库，生成真正可运行的 `main.py`。
- **S12 冒烟测试**：用 `SMOKE_TEST=1` 快速验证代码不崩溃。LLM 作为 Agent 自动诊断错误、修复代码、重新运行——真正的 **Agentic Debug**。
- **S14 正式实验**：在 GPU 沙箱中执行完整实验，LLM 监控执行过程，处理 OOM、NaN、路径错误等运行时问题。
- **S15 迭代优化**：基于基线结果反复 **编辑-运行-评估**，逐步提升实验指标。

### Phase 4 · 分析决策（S16-S17）

分析实验结果后，系统做出研究决策：**PROCEED**（继续写论文）、**PIVOT**（方向不对，回退重新假设）、**REFINE**（需要迭代优化）。这让整个流程具备了**自我纠偏**能力。

### Phase 5 · 论文输出（S18-S26）

自动生成论文大纲、撰写全文、经过内部"同行评审"修改后，通过最终质量门控，输出完整论文 + 代码 + 图表 + 实验日志。

## 三、三个核心技术亮点

### 1. Claw-Code Turn Loop — 让 LLM 像工程师一样写代码

传统的 LLM 代码生成是「一次性输出」，写错了就没办法。Claw 的 Turn Loop 是一个**迭代式工具调用循环**：

```
LLM 思考 → 调用工具（bash / read_file / edit_file / grep_search）
         → 观察结果 → 继续思考 → 再调用工具 → ...
```

LLM 可以**读代码、跑命令、看报错、改代码、再跑**——和真实工程师的调试循环一模一样。每一轮对话都会被完整记录为 Trace，方便回溯和审计。

### 2. Harness 信任边界 — 实验结果不可伪造

Claw 注入了一个 **Experiment Harness**（实验装具），作为 LLM 不可编辑的信任边界：

- **标准化输出**：所有实验必须通过 Harness 输出 `results.json`，格式统一、可解析
- **认识论诚实**：系统级规则禁止 LLM 伪造指标、硬编码结果、用文件名充当标注
- **Metric 验证**：自动检查 NaN、Inf、数值范围，确保结果来自真实计算

### 3. PRM 质量门控 — 过程奖励模型做质控

在关键阶段（文献筛选 S5、实验设计 S9、质量门控 S23），系统引入 **Process Reward Model（过程奖励模型）** 作为 LLM-as-Judge：

- 多次独立评分 + 投票机制（默认 3 票）
- 评分 -1 / 0 / +1，负分触发**自动重试**
- 结合 **Human-in-the-Loop**：关键门控支持人工审批，拒绝可触发回滚到指定阶段

## 四、三种研究模式

| 模式 | 适用场景 | 说明 |
|------|---------|------|
| **Explore** | 开放性探索 | 给一个研究方向，系统自动发现最有价值的课题并执行 |
| **Discussion** | 多智能体辩论 | 多个 Agent 各自持立场，通过结构化辩论收敛到最优方案 |
| **Reproduce** | 论文复现 | 上传 PDF，系统自动提取方法细节并复现实验 |

## 五、为什么我们需要这样的系统？

当前 AI 辅助科研工具大多停留在「帮你写代码」或「帮你总结论文」的单点能力。但真实的科研是一个**高度迭代、多阶段、需要判断力**的过程：

- 文献调研可能发现方向不对，需要**回退**
- 代码第一次跑肯定有 bug，需要**反复调试**
- 实验结果可能不理想，需要**迭代优化**
- 写完论文可能发现逻辑漏洞，需要**重新分析**

Claw AI Lab 的设计哲学是：**不做一次性的自动化，做一个能自我纠偏的研究闭环。** 每个阶段都有质量门控，关键节点保留人工介入的能力，让 AI 成为你的研究团队，而不是一个不可控的黑盒。

## 六、快速体验

```bash
# 克隆项目
git clone https://github.com/Claw-AI-Lab/Claw-AI-Lab.git

# 配置你的 LLM API 和 GPU
cp examples/config_template.yaml backend/runs/project_configs/my-project.yaml
# 编辑 YAML：填入 api_key、base_url、数据集路径等

# 启动服务
python backend/services/agent_bridge.py
```

打开 Web UI，输入你的研究想法，点击开始。剩下的，交给 Claw。
